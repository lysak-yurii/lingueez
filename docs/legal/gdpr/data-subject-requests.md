# Handling Data-Subject Requests (DSARs)

*Internal procedure — Articles 15–22 GDPR.*

**Last reviewed:** 2026-06-23

Most rights are **self-service in the app**; this procedure covers requests sent by email.

## Intake
- Channel: **privacy@lingueez.app**.
- Verify identity (e.g. confirm from the account's registered email) before acting, but don't
  demand excessive data.
- **Respond within one month** (extendable by two months for complex requests — tell the user).
- Requests are **free** unless manifestly unfounded or excessive.

## Rights and how to fulfil them

| Right | Article | How |
|-------|---------|-----|
| **Access** | 15 | Point the user to **Export**; or generate a copy of their account + synced rows. |
| **Portability** | 20 | **Export** provides an open, machine-readable copy. |
| **Rectification** | 16 | User edits in-app; or correct on request. |
| **Erasure ("right to be forgotten")** | 17 | **Delete account** removes account + all synced content. Confirm cascade across `words/texts/tags/word_tags`. |
| **Restriction** | 18 | Disable sync / flag the account; describe how done. |
| **Objection** | 21 | Stop the objected-to processing (e.g. turn off optional features). |
| **Withdraw consent** | 7(3) | User disables optional online features in Settings. |

## Records
- Log each request: date received, identity check, action taken, date completed.

## Notes
- For data sent to optional third parties (OpenAI/Google/DeepL) using the user's **own** API
  key, direct the user to that provider; explain the boundary in the response.
