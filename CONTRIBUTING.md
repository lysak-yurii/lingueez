# Contributing

Thanks for your interest in improving Lingueez! Contributions of all kinds are
welcome — bug reports, fixes, features, and documentation.

## License of contributions

This project is released under the **GNU Affero General Public License v3.0**
(see [`LICENSE.txt`](LICENSE.txt)). You are free to fork, modify, and redistribute
it under those terms.

Because the project has a single maintainer who remains its owner, every
contribution must be made under the **Contributor License Agreement** in
[`CLA.md`](CLA.md). The CLA lets the maintainer keep the project relicensable and
legally clear, while you retain copyright to your own work.

## How to contribute

1. **Open an issue first** for anything non-trivial, so we can agree on the approach
   before you invest time.
2. **Fork** the repository and create a topic branch for your change.
3. **Keep changes focused** — one logical change per pull request.
4. **Match the existing style** of the surrounding code.
5. **Sign off your commits** to certify the CLA (see below).
6. **Open a pull request** describing what changed and why.

## Agreeing to the CLA

By signing off each commit, you certify that you have read and agree to
[`CLA.md`](CLA.md). Add a sign-off line with:

```bash
git commit -s -m "Your message"
```

This appends a `Signed-off-by: Your Name <your@email>` trailer to the commit.

## Reporting bugs

Open an issue with clear steps to reproduce, what you expected, what happened, and
your operating system.
